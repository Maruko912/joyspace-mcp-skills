# 月报总结

基于 JoySpace 原始月报文档，提炼出简洁、有逻辑、突出重点的月报总结。适合在办公软件中发送给领导。

## 前置条件

- 用户提供原始月报的 JoySpace 文档 URL
- Chrome DevTools MCP 已配置并连接

## 使用方式

```
/月报总结 <JoySpace文档URL>
```

## 操作流程

### 步骤 1：提取原文

使用 `/joyspace-read` skill 的流程，完整提取文档所有文本内容。

> **铁律：1:1 保留原文。** 后续所有总结、改写都必须基于原文逐条核实。不在自己的总结版本上反复迭代。

### 步骤 2：识别核心数据

从原文中提取所有关键数字，**原样记录，不做任何换算或改写**：

- 止损金额、追损金额、案件金额
- 拦截数量、识别量、覆盖率
- 效率提升百分比、成本降低百分比
- 目标达成率

### 步骤 3：按模板撰写总结

#### 总结模板

```
<N>月市场止损X万，案件打击新增X万，追损X万，……，目标总体达成X%。

一、<板块名称>
- <做了什么>：<结果/价值/目的>
- <做了什么>：<结果/价值/目的>

二、<板块名称>
- ...

三、<板块名称>
- ...

四、<板块名称>
- ...
```

#### 模板规则

1. **开头一句话**：汇总最核心的 3-5 个数字（止损、案件、追损、达成率等），让领导一眼看到全貌
2. **板块数量**：精简到 3-5 个，砍掉纯技术向、无业务影响的板块（如"系统稳定性与治理"）
3. **每条必须有"做了什么→结果/价值/目的"**：
   - 不能只写"做了XX"而不解释为什么做、效果如何
   - 不能只写"XX方案设计完成"而不说这个方案解决什么问题
   - 领导要知道：你为什么做这件事，它带来了什么价值
4. **砍掉的内容**：
   - 过程细节（"完成开发、联调、测试"→ 简化为"从0到1建成"）
   - 技术实现细节（pfinder升级、系统拆分等，除非有直接业务影响）
   - 还在规划/对接中的低优先级事项
   - 发现过程描述（"情报披露发现"这类模糊表述直接去掉）
5. **保留的内容**：
   - 所有涉及金额的数据
   - 重大案件（名称 + 作弊类型 + 金额 + 状态）
   - 从0到1的能力突破
   - 有量化结果的效率提升
   - 有实验数据支撑的结论
6. **领导可读性 — 消除内部术语盲区**：
   - 读者是了解行业但不了解团队内部细节的领导
   - **可以保留的**：行业通用术语（RTA、CPS、SDK、ROI等），领导在广告营销领域能理解
   - **必须处理的**：内部项目代号、策略编号、系统名称（如"419策略"、"Onetrans"、"pfinder"），这些只有团队内部才懂
   - 处理方式：用通俗描述替代，如"CPS 419策略完成重构" → "CPS反作弊核心策略完成重构"
   - 检查标准：一个了解广告营销但没参加过你们周会的领导，能看懂每一条吗？

### 步骤 4：逐条核实

将写好的总结与原文**逐条对照**，检查：

- [ ] 所有数字是否与原文一致（金额、百分比、数量）
- [ ] 专有名词是否准确（产品名、案件名、渠道名）
- [ ] 状态描述是否准确（"已完成" vs "调研中" vs "规划中"）
- [ ] 没有添加原文中不存在的信息
- [ ] 没有改变原文的因果关系或含义

> **常见错误示例（必须避免）**：
> - 原文"调研Onetrans模型" → 错写为"完成架构设计"（夸大状态）
> - 原文"广义新用户占比高出11.2%" → 错写为"ROI提升11.2%"（偷换概念）
> - 原文"约8%设备天御有标记但JDG未标记" → 错写为"准确率领先8%"（曲解含义）
> - 原文"预估日扩充2500万" → 错写为"新增覆盖2500万"（混淆预估与已完成）

### 步骤 5：输出给用户确认

将总结以文本形式输出给用户review，**不要直接写入文档**。等用户确认或提出修改意见后，再根据反馈调整。

如果用户要求修改，**回到原文重新核实**修改部分，不要在当前总结版本上直接改。

### 步骤 6：写入 JoySpace 文档

用户确认后，使用 `/joyspace-create` skill 创建新文档写入。

## 关键注意事项

1. **数据准确是第一优先级**：宁可少写一条，不能写错一个数字
2. **不要在自己的总结上迭代**：每次修改都回到原文核对，避免越改越偏
3. **每条要有价值感**：领导看月报要知道"你做了什么、为什么做、效果如何"，不是技术流水账
4. **先确认再写入**：总结先给用户看，确认后再创建文档，避免反复新建
5. **板块命名要体现价值**：用"重大案件发现与打击"而不是"案件工作"，用"流量拦截与识别能力提升"而不是"模型策略"
